# NodeTSExample

A minimal Node.js HTTP server written in TypeScript, with no web framework — only the built-in `http`, `fs`, and `url` modules. Useful as a starting point for understanding how Node serves HTTP requests at the lowest level before reaching for Express or similar frameworks.

## What it does

- Starts an HTTP server on `0.0.0.0:13000` (or `127.0.0.1:5500` if `DEPLOY` is `false` in [ts/server.ts](ts/server.ts)).
- On every request, reads [index.html](index.html) from disk, substitutes the `{{ name }}` placeholder with the `name` query-string parameter (or `"Anonyme"` if absent), and returns the result.
- Logs each connection with a UTC timestamp produced by [ts/datetime.ts](ts/datetime.ts).
- When `DEPLOY` is `true`, it queries an external service via [ts/WhatsMyIpAddress.ts](ts/WhatsMyIpAddress.ts) and prints the public IP under which the server is reachable.

Open `http://localhost:13000/?name=Alice` in a browser and you should see a page greeting Alice.

## Project layout

```
NodeTSExample/
├── index.html              # Template served on every request, contains {{ name }}
├── package.json
├── tsconfig.json
├── ts/                     # TypeScript sources (input)
│   ├── server.ts           # HTTP server entry point
│   ├── datetime.ts         # GetCurrentDateTime() helper
│   └── WhatsMyIpAddress.ts # Public-IP lookup helper
└── js/                     # Compiled JavaScript (output, generated by tsc)
```

## Requirements

- Node.js 18 or newer (tested on Node 24).
- npm.

## Install

```bash
npm install
```

## Build

Compile `ts/*.ts` to `js/*.js`:

```bash
npm run build
```

## Run

After building:

```bash
npm start
```

Then open `http://localhost:13000/?name=YourName`.

## Develop

Run TypeScript in watch mode and restart the server on every change:

```bash
npm run dev
```

This uses `concurrently` to run `tsc -w` and `nodemon js/server.js` in parallel.

## Configuration

The only knob is the `DEPLOY` constant near the top of [ts/server.ts](ts/server.ts):

| `DEPLOY` | Hostname            | Port   | Public-IP lookup |
| -------- | ------------------- | ------ | ---------------- |
| `true`   | `0.0.0.0`           | `13000`| yes              |
| `false`  | `127.0.0.1`         | `5500` | no               |

Set it to `false` for purely local development; leave it as `true` to bind on all interfaces (don't forget to open the port on your firewall/router).
