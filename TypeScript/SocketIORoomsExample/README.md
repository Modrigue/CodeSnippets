# Socket.io Rooms GUI Example

A small demo showing how to use [Socket.IO](https://socket.io/) **rooms** to build a multi-room, multi-player game lobby. Players connect from a browser, then either **create** a new room (optionally password-protected) or **join** an existing one. Each room has its own player list, configurable size, and game state — broadcast events are scoped to the room they belong to.

The sample is intentionally small: the "game" itself is a stub. The interesting part is the lobby flow — room creation, joining, password checks, capacity limits, player parameters, kick handling, and creator-leaves-cleanup — all driven through Socket.IO events.

## Features

- **Create / join rooms** — name, optional password, configurable max number of players, configurable number of rounds.
- **Per-room broadcast** — `io.to(room).emit(...)` so events only reach players inside that room.
- **Password-protected rooms** — checked server-side on join.
- **Capacity enforcement** — joining a full room is rejected.
- **Player parameters** — name, color, team, ready flag, synchronized to all room members.
- **Creator semantics** — when the room creator disconnects during setup, the room is dismissed and all players are kicked.
- **Live rooms list** — broadcast to every connected client whenever a room is created, updated, or removed.
- **Kick** — the creator can kick a player from a room.

## Architecture

```
┌──────────────────────────────┐         ┌──────────────────────────────┐
│  Browser                     │         │  Node.js + Express           │
│                              │         │                              │
│  index.html                  │         │  ts/server.ts                │
│   ├─ js/interface.js  (DOM)  │ ◀──────▶│   - Socket.IO server         │
│   ├─ js/client.js     (sock) │  WS     │   - Room registry (Map)      │
│   └─ js/pages/*.js    (UI)   │         │   - Player registry          │
│                              │         │   - Event handlers           │
└──────────────────────────────┘         └──────────────────────────────┘
```

Both the client-side scripts and the server are written in TypeScript and share a common project. `tsc` compiles `ts/**/*.ts` to `js/**/*.js`; the server is run from `js/server.js` and the browser loads the other compiled files via `<script>` tags.

## Socket.IO events

| Direction       | Event                | Payload                                              | Purpose                                                    |
| --------------- | -------------------- | ---------------------------------------------------- | ---------------------------------------------------------- |
| client → server | `createNewRoom`      | `{ name, room, password }`                           | Create a new room and become its creator.                  |
| client → server | `joinRoom`           | `{ name, room, password }`                           | Join an existing room (password-checked, capacity-checked). |
| client → server | `setRoomParams`      | `{ nbPlayersMax, nbRounds }`                         | Update room parameters during setup.                       |
| client → server | `setPlayerParams`    | `{ color, team, ready }`                             | Update self-parameters; broadcast to room members.         |
| client → server | `play`               | `{}`                                                 | Start the game (creator) or join a started game.           |
| client → server | `kickPlayer`         | `{ id }`                                             | Kick a player from a room.                                 |
| server → client | `roomsList`          | `[{ room, nbPlayersMax, nbPlayers, status }, ...]`   | Live list of rooms (broadcast to all clients).             |
| server → room   | `updatePlayersList`  | `[{ id, name }, ...]`                                | Updated player list for a room.                            |
| server → room   | `updateRoomParams`   | `{ room, nbPlayersMax, nbRounds }`                   | Updated room parameters.                                   |
| server → room   | `updatePlayersParams`| `[{ id, name, color, team, ready }, ...]`            | Updated player attributes.                                 |
| server → room   | `kickFromRoom`       | `{ room, id }`                                       | Tell a specific player (or all if `id == ""`) to leave.    |
| server → room   | `playGame`           | `{ room, nbPlayersMax, nbRounds }`                   | Game starts (or new player joins started game).            |

## Requirements

- Node.js 18 or newer (tested on Node 24).
- npm.

## Install

```bash
npm install
```

## Build

Compile `ts/**/*.ts` to `js/**/*.js`:

```bash
npm run build
```

## Run

After building:

```bash
npm start
```

Then open `http://localhost:13000/` in two or more browser windows to simulate multiple players.

## Develop

Run TypeScript in watch mode and restart the server on every change:

```bash
npm run dev
```

This uses `concurrently` to run `tsc -w` and `nodemon js/server.js` in parallel.

## Configuration

The `DEPLOY` constant at the top of [ts/server.ts](ts/server.ts) toggles between two modes:

| `DEPLOY` | Behaviour                                                                            |
| -------- | ------------------------------------------------------------------------------------ |
| `true`   | Express serves static files (HTML, CSS, JS) from the project root, Socket.IO is attached to the same HTTP server, port = `process.env.PORT` or `13000`. |
| `false`  | Socket.IO listens on its own port (`5500`), Express is unused for static files (server-only mode for testing). |

For normal browser-based use, leave `DEPLOY = true`.

## Known issue: client / server Socket.IO version mismatch

[index.html](index.html) currently loads the Socket.IO **v2.4.0** client from a CDN, while the server uses Socket.IO **v4.x**. Socket.IO v2 and v4 are **not wire-compatible** — connections will fail. To make the demo actually work, replace the CDN script tag in `index.html` with a v4 client, e.g.:

```html
<script src="https://cdn.socket.io/4.8.1/socket.io.min.js"></script>
```

or serve the client bundle that ships inside `node_modules/socket.io/client-dist/`.
