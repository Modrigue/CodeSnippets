# ExpressTSExample

A minimal [Express 5](https://expressjs.com/) HTTP server written in TypeScript. Demonstrates the basics of routing, request body parsing, and session handling without any extra abstractions — useful as a starting point or reference for an Express + TypeScript project.

## What it does

- Starts an Express server on `0.0.0.0:13000` (or `127.0.0.1:5500` if `DEPLOY` is `false` in [ts/server.ts](ts/server.ts)).
- Serves [index.html](index.html) on `GET /`. The page contains a small form with a textarea and a submit button.
- On `POST /`, reads the `word1` field from the form, logs it to the console along with the client IP, and re-serves the same page. If the field is empty, an error message is stored on the session (`request.session.error`) so it can be surfaced on the next request.
- Logs each connection with a UTC timestamp produced by [ts/datetime.ts](ts/datetime.ts).
- When `DEPLOY` is `true`, it queries an external service via [ts/WhatsMyIpAddress.ts](ts/WhatsMyIpAddress.ts) and prints the public IP under which the server is reachable.

Open `http://localhost:13000/` in a browser, type a word, hit **Send**, and watch the server log it.

## Middleware stack

| Middleware                         | Purpose                                       |
| ---------------------------------- | --------------------------------------------- |
| `express.urlencoded({ extended: false })` | Parse `application/x-www-form-urlencoded` form bodies |
| `express.json()`                   | Parse `application/json` request bodies       |
| `express-session`                  | In-memory session storage (cookie `secure: false` — HTTP only) |

> Note: `express.json()` and `express.urlencoded()` are now built into Express, so the standalone `body-parser` package is no longer required.

## Project layout

```
ExpressTSExample/
├── index.html              # Page served on GET / (and re-served after POST /)
├── package.json
├── tsconfig.json
├── ts/                     # TypeScript sources (input)
│   ├── server.ts           # Express server entry point: routes + middleware
│   ├── datetime.ts         # GetCurrentDateTime() helper
│   └── WhatsMyIpAddress.ts # Public-IP lookup helper
└── js/                     # Compiled JavaScript (output, generated by tsc)
```

## Requirements

- Node.js 18 or newer (tested on Node 24).
- npm.

## Install

```bash
npm install
```

## Build

Compile `ts/*.ts` to `js/*.js`:

```bash
npm run build
```

## Run

After building:

```bash
npm start
```

Then open `http://localhost:13000/`.

## Develop

Run TypeScript in watch mode and restart the server on every change:

```bash
npm run dev
```

This uses `concurrently` to run `tsc -w` and `nodemon js/server.js` in parallel.

## Configuration

The only knob is the `DEPLOY` constant near the top of [ts/server.ts](ts/server.ts):

| `DEPLOY` | Hostname    | Port    | Public-IP lookup |
| -------- | ----------- | ------- | ---------------- |
| `true`   | `0.0.0.0`   | `13000` | yes              |
| `false`  | `127.0.0.1` | `5500`  | no               |

Set it to `false` for purely local development; leave it as `true` to bind on all interfaces (don't forget to open the port on your firewall/router).

The session secret (`'keyboard cat'`) and `cookie.secure: false` are hard-coded for demonstration. **Do not reuse those values in production** — set a strong secret from an environment variable and enable `secure: true` behind HTTPS.
